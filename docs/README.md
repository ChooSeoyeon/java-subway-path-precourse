# 부록

## 기능 요구사항 분석

- 초기 설정
    - 역, 노선, 구간 정보(양의 정수 km,분) 초기설정
- 경로 조회 기능
    - 출발역과 도착역을 입력받기
    - 경로 조회하기 (최단 거리, 최소 시간)
    - 경로 조회 결과로 총 거리, 총 소요 시간 출력하기
- 예외 처리
    - 출발역과 도착역이 같으면 에러
    - 출발역과 도착역이 연결되어 있지 않으면 에러 (같은 노선에 등록되어 있지 않다면!)
    - 그 외 정상적으로 프로그램이 수행되지 않은 경우 에러

## 프로그래밍 요구사항 분석

- model
    - Station(지하철역)
    - Line(노선)
- repository
    - StationRepository
    - LineRepository
- 최단 경로 라이브러리
    - jgraph 라이브러리로 최단 경리 간편하게 조회 가능
    - Dijkstra 알고리즘 미션에 간단하게 적용
        - 정점 : 지하철역
        - 간선 : 지하철역 연결정보
        - 가중치 : 거리 혹은 시간

## 스토리텔링식 설계

- 이야기 :
- 협력 : 요청과 응답
- 책임 : 하는 것(요청, 계산)과 아는 것
- 역할 : 책임의 집합(재사용성)
